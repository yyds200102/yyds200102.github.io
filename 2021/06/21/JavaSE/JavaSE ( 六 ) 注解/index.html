JAVASE(六):注解

发表于2021-06-21|更新于2021-06-21|Java
|字数总计:1.2k|阅读时长:4分钟|阅读量:

1、概述

  • JDK5.0开始引入
  • 作用类似于注释，可以对程序作出解释，这些信息可以在编译、类加载、运行时被读取
  • 通过使用 Annotation, 程序员可以在不改变原有逻辑的情况下, 在源文件中嵌入一些补充信息，现在的主流Java框架均大量使用注解，可以这么说：框架 = 注解 + 反射 + 设计模式

2.2、元注解：

​ 负责注解其他注解，为meta-annotation类型，一共有4个，通过元注解就可以自定义注解，上面提到的Java内置注解其实就是Java预先用元注解帮我们定义好的，基本的4个元注解的作用如下

  1. @Target：描述注解的使用范围
  2. @Rentention：描述该注解的生命周期，即在什么时候有效
  3. @Document：说明该注解将被包含在Javadoc中
  4. @Inherited：说明子类可以继承父类中的注解

@Target、@Rentention都是有参数的，各参数的定义如下

  • TYPE : 类、接口或enum声明
  • FIELD: 域(属性)声明
  • METHOD: 方法声明
  • PARAMETER: 参数声明
  • CONSTRUCTOR: 构造方法声明 LOCAL_VARIABLE:局部变量声明
  • ANNOTATION_TYPE:注释类型声明
  • PACKAGE: 包声明
  • SOURCE:在源文件中有效（即源文件保留），编译器直接丢弃
  • CLASS:在class文件中有效（即class保留） ， 当运行 Java 程序时, JVM不会保留注解
  • RUNTIME：在运行时有效（即运行时保留），当运行 Java程序, JVM会保留注释，此时可以通过反射获取注解。

3.2、分析

注解的格式： public @interface 注解名 { 定义体 } 并且再加上元注解

这里我们先：

  1. 使用 @Retention(RetentionPolicy.RUNTIME) 表明 该注解是在 运行时 有效的
  2. 使用 @Target(ElementType.TYPE) 表明该接口定义在 类、接口或enum上
  3. 指定名称为 MyAnnotation

接着定义方法体的内容：

  • String age() default “20”; 表明定义了一个参数，名字为age，类型为String，默认值为20(可以不用默认值)

使用：

  • @MyAnnotation(age = “30”) 使用注解，给参数age赋值30并且标记在test类上面

注意点

  • 类型只能是基本数据类型、String类型、Class类型、enum类型、Annotation类型、以上所有类型的数组
  • 没有成员定义的 Annotation 称为标记; 包含成员变量的 Annotation 称为元数据 Annotation
  • 如果只有一个注解，可以这么定义：String value() default “20” ，这样使用时可以@MyAnnotation(“30”)

4、Java8新增特性

4.1、可重复注解

​ 在 Java 8 之前我们不能在同一个类、同一方法上重复使用同一个注解，在 JDK 8 中引入了一个注解 @Repeatable 来标识某个注解是可被重复使用的，但是需要一个容器注解

4.2、类型注解

DK 8 为 @Target 元注解新增了两种类型：TYPE_PARAMETER、TYPE_USE

  1. TYPE_PARAMETER：表示该注解能够写在 声明为 泛型 的语句中，类型参数声明如：、<T extends 某个类>

  2. TYPE_USE：表示注解可以用在[任何用到类型]的地方使用
