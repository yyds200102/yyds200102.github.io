1、集合框架概述

1.1、简介

在学习Java的集合框架之前，都是通过数组来存放数组，但是数组有一些弊端

  • 数组初始化以后，长度就确定了
  • 数组中提供的属性和方法少，不便于进行添加、删除、插入等操作
  • 存储数据的特点单一，即都是有序可重复的

Java集合类可以弥补数组在存储数据方面的不足

1.2、分类

Java 集合可分为 Collection 和 Map 两种体系

Collection接口：单列数据，定义了存取一组对象的方法的集合

  • List：元素有序、可重复的集合
  • Set：元素无序、不可重复的集合

Map接口：双列数据，保存具有映射关系“key-value对”的集合

3.2、List实现类之一：ArrayList

本质：

  • JDK1.7：ArrayList像饿汉式，直接创建一个初始容量为10的数组

4、Collection子接口：Set

  • Set接口是Collection的子接口，set接口没有提供额外的方法
  • Set 集合不允许包含相同的元素，如果试把两个相同的元素加入同一个，会失败

4.1、Set实现类之一：HashSet

特点：

  1. 按 Hash 算法来存储集合中的元素，因此具有很好的存取、查找、删除性能
  2. 无序、线程不安全、集合元素可以为null
  3. HashSet 是 Set 接口的典型实现，大多数时候使用 Set 集合时都使用这个实现类
  4. 底层是数组+散列算法，也就是说Hashset是通过数组存放数据的

注意点：

​ 若是在Set容器中存放对象，则对象的那个类必须要重写 equals 方法和 hashcode 方法，即保证equals相等的对象，hashcode也必须相等，以实现对象相等规则。相等的对象必须具有相等的散列码。

添加元素的过程

  1. 调用 方法获取哈希值，并根据哈希值判断通过散列函数判断在底层数组的存储位置。
  2. 判断 hashcode 值是否已经存在，若已经存在，则再通过 equals 方式判断
  3. 若 equals 返回 true ，则添加失败；若为false ，则通过链表的方式继续链接，保证唯一性

底层结构：

​ 底层是数组，初始容量为16，当如果使用率超过0.75，就会扩大容量为原来的2倍

HashSet扩展:LinkedHashSet

特点：

  1. LinkedHashSet 是 HashSet 的子类，不允许元素重复，
  2. LinkedHashSet 根据元素的 hashCode 值来决定元素的存储位置，但它同时使用双向链表按照插入顺序维护元素的次序
  3. LinkedHashSet插入性能略低于 HashSet，但是迭代访问性能好一些

3、有序的，查询速度比List快

4、通过compare判断重复与否，只要返回的不是0，就可以增添，不再通过hashcode

5、Map接口

概述：

  1. Map与Collection并列存在。用于保存具有映射关系的数据:key-value
  2. Map 中的 key 和 value 都可以是任何引用类型的数据
  3. Map 中的 key 用Set来存放，不允许重复（重复则会覆盖值），value可以重复

常用方法

迭代Map集合

  • 通过Map.keySet()遍历key和value
  • 通过Map.entrySet()遍历集合

5.1、Map实现类之一：HashMap

特点：

  1. 允许使用null键和null值，与HashSet一样，不保证映射的顺序。

  2. key构成的集合是Set:无序的、不可重复的，重写：equals()和hashCode()

  3. 判断Key相等：equals和hashcode value只通过equals判断

  4. 哈希表的大小为 2 的次方，会根据传来的参数调整，但是无论如何都是2的次方

底层结构：

  1. JDK 7及以前版本：HashMap是数组+链表结构(即为链地址法)
  2. JDK 8版本发布以后：HashMap是数组+链表+红黑树实现。

原理探究(JDK1.8)：

  1. 实例化一个HashMap时，会初始化initialCapacity（默认容量）和loadFactor（填充因子）
  2. put数据时，才会根据默认容量（JDK8为16）创建Node数组，数组存放数据的地方成为bucket（桶），每个bucket都有自己的索引，所以查找速度非常快
  3. 每个bucket存一个Node对象，里面含有指向下一个元素的next引用变量，所以一个bucket可能形成Node链，或者是TreeNode树，而新添加的元素作为链表的last，或树的叶子结点

哈希函数：

  • JDK7：通过扰动函数处理过后得到 hash值，然后通过 (n - 1) & hash判断当前元素存放的位置
  • JDK8：优化了代码

扩容情况：

  1. HashMap的元素个数大于 initialCapacity（默认容量）* loadFactor（填充因子）时，就会扩大一倍。

  2. 扩容时会重新计算每个元素的位置，消耗资源，所以可以预设大小，减少资源消耗

树形化：

​ 若是一个 Node 链 里面的元素超过了8个（容量小于64则会优先先尝试扩大容量减少哈希冲突），就会尝试树形化，然后数节点由Node类型变为TreeNode类型，当然，映射关系移除后，若小于6个，重新变回Node链

​ 也就是说，桶里的Node链 > 8 ，hashMap 容量 > 64时，实际上才会发生树形化

loadFactor（填充因子）的理解

  1. 负载因子越大，数据密度越大，发生碰撞的几率越高，数组中的链表越容易长，查询或插入时的比较次数多，性能下降
  2. 负载因子越小越容易扩容，容易浪费空间
  3. 最好设置为0.7~0.75

  • LinkedHashMap 可以维护 Map 的迭代顺序：与迭代顺序插入顺序一致

5.1.2、HashMap扩展：IdentityHashMap

重新定义了hashcode的取值方法：System.identityHashCode(Object obj)

  • 实际调用的是Object的 hashcode方法，比较地址
  • 两个对象比较相等是不再使用 equals 而是使用 ==
  • 也是就说，只要是两个对象，永远不会相等

5.1.3、HashMap扩展：WeakHashMap

  • 使用的是弱引用，随时可能被回收

  • 用途：缓存等

7、Collections工具类：增强集合

7.1、同步集合

Collections 类中提供了多个 synchronizedXxx() 方法，该方法可使将指定集合包装成线程同步的集合，从而可以解决多线程并发访问集合时的线程安全问题

7.2、小集合

Java9引入静态方法，可以生成给定元素的集或者列表，在Java8相当于Arrays.asList

  • 注意这些也是固定大小，而且不能增添、删除、修改内容

7.3、子集合

  • 可以把集合分解成子集合
  • 注意子集合并没有和父集合分离，子集合上的操作会在父集合上同步执行

7.4、不可修改的视图

这些视图对现有集合增加了一个运行时检查，如果发现修改了，就抛出一个异常

7.5、严格检查集合

  • 编译器有时候对于集合中添加错误的情况，没能指示出来
  • 这样定义之后就会进行严格的检查
