1、线程池概述

1.1、概念

不建议用： new Thread().start();

  • 浪费服务器的资源，创建和销毁 浪费资源

  • 不能立马就销毁对象。大量的对象，GC

  • 线程的生命周期比较长，不好管理线程的状态，中断….

  • 不支持周期，调度的功能。线程大容易竞争资源

线程池：维护一批 new thread().start() 放到一个池子或者队列集合里面。提交任务就取出来一条线程直接使用，速度快，结束任务后，把线程返回线程池。或者空闲的线程销毁掉。

优点：

  • 不用频繁创建和销毁线程。减少浪费资源。

  • 便于管理线程的状态。

  • 支持周期性，提高相应速度。减少资源竞争，

  • 便于控制线程的数量

2、Executor框架简介

2.1、简介

Executor框架的结构

  • 任务：即线程，需要实现Runnable或者callable接口
  • 任务的执行：任务执行的核心接口 Executor 及实现类 ExecutorService（其中ThreadPoolExecutor 和 ScheduledThreadPoolExecutor 实现了该接口）
  • 异步计算的结果：包括 Future 接口和其实现类 FutureTask

任务执行的过程

  1. 创建 ExecutorService 对象，即线程池
  2. 把 Runnable 对象或者 Callable 对象交给 ExecutorService 对象执行
  3. 如果是 Callable 线程，返回一个 Future 对象的结果，当然也可以直接创建 FutureTask 对象 交给 ExecutorService 对象执行。
  4. 通过 Future 或者 FutureTask 对象 获得结果

2.2、 Executor 对象的主要成员

  • Executor接口：Executor框架的基础，实现任务的提交和执行分离
  • ExecutorService：一个线程池接口，继承了Executor接口，一般用作 具体的线程池 向上转型成该类对象。
  • ThreadPoolExecutor：线程池的核心实现类，用于创建线程池，通常使用工厂类 Executors 来创建（也可以自定义）。继承了AbstractExecutorService类，AbstractExecutorService 继承了 ExecutorService接口。
  • ScheduledThreadPoolExecutor：线程池的一个实现类，用于创建线程池
  • Future接口：线程池执行Callable 任务后，返回的结果为 Future类型

3、创建线程池

3.1、简介

  • ThreadPoolExecutor 是 线程池的核心实现类，提供了 3种线程池的实现
  • ScheduledThreadPoolExecutor 继承了 ThreadPoolExecutor，主要用于给定延迟之后运行任务或者定期执行任务。

3.2、JDK四大内置线程池

  • SingleThreadExecutor：单个线程池
  • FixedThreadPool：自己控制线程数量
  • CachedThreadPool：根据需要自动决定容量的线程池
  • ScheduledThreadPool：可以周期性和延迟执行线程

创建线程：通过Executors工具类

ScheduledThreadPool：可以使用**schedule启动**线程周期性运行

通用方法：都可以使用execute和submit启动线程，其中submit支持callable线程

  • excute ：无返回值
  • submit ：有返回值

3.3、自定义线程池

  • 上述几个线程池都是参数不同的ThreadPoolExcutor方法，
  • 该方法有七个参数

七大参数

  1. 初始线程池大小
  2. 线程池最大大小
  3. 等待超时时间大小，超过多少时间没有线程进入就把线程池的大小变为**初始值**
  4. 第3个参数的时间单位 （如TimeUnit.SECONDS）
  5. 阻塞队列
    • 线程池满就让线程进入队列，
    • 队列第一次满(初始线程大小和队列满了)，就把线程池变为最大的大小，
    • 队列第二次满(线程最大大小和队列都满)，就执行拒绝策略
  6. 线程工厂，默认即可
  7. 拒绝策略,

4、FutureTask详解

概念：FutureTask继承了 Future类，实现类 Runnable 接口，有多种状态

作用：

  1. 作为 线程池执行的返回值，通过 get 方法获取结果
  2. 把线程作为参数填入 FutureTask 对象中，这样 ExecutorService 对象 submit 的就不是线程而是 FutureTask 对象，
  3. 通过· cancel 取消任务。
  4. 作为一个线程单独执行。
  • FutureTask实例

cancel方法详解

  • 参数为true：尝试把通过中断来终止当前线程
  • 参数为false：尝试把线程状态设置为CANCELLED
