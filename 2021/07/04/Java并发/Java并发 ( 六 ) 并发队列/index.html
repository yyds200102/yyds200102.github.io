1、并发队列Queue的分类和简介

分类：

  • 阻塞队列：提供不同的方法，添加、移除效果会根据API的不同而不同；每个方法都会使用锁让每次只能有一个线程操作队列。
  • 非阻塞队列：队列的容量是无线大的。Integer.MaxValue(); 基于CAS ,没有使用锁，所以并发能力都是强，数据不需要等待。

堵塞：调用await让线程堵塞，等队列有空间或者数据了再唤醒。

特点：

  • 阻塞队列：ReentrantLock【悲观策略】，可以有两把锁（入队和出队不同的锁）或者一把锁
  • 非阻塞队列【CAS冲突监测的乐观锁机制】

注意：堵塞队列都继承了AQS、BlockingQueue

堵塞队列通用API：

方法 抛出异常 有返回值，不抛异常 阻塞 超时等待
添加 add offer put offer
移除 remove poll take poll
监测队首元素 element peek

由 head 节点和 tail 节点组成，每个节点由节点元素 item 和指向下一个节点 next 的引用组成，初始情况情况 head为空，tail 节点等于head节点。

3、阻塞队列-ArrayBlockingQueue

继承了BlockingQueue接口，底层是数组结构，按照先进先出的顺序

  • 默认情况下是非公平的：堵塞的线程会被全部唤醒，会同时竞争，所以不一定先堵塞的先访问队列，可以设置为公平的（参数加true即可）
  • 创建需要需要指定大小

5、阻塞队列-PriorityBlockingQueue

结构：二叉堆+数组+CAS,无界队列

特点：

  • 默认按照升序排列，需要实现排序接口或者传入排序器。
  • 默认是11个大小容量，可以自动扩容到无界队列MAX_VALUE

扩容原理：

  1. 获取锁之后，如果需要扩容就会先释放锁，为了性能：先让其他线程执行操作
  2. 使用CAS控制只有一个线程去扩容，失败的线程调用Thread.yield()让出cpu让出时间片。
  3. 创建新数组，创建之后获取锁，进行复制操作。

6、阻塞队列-DelayQueue

延迟队列：可以看做特殊优先级队列，只比较剩余时间

  • 添加元素可以指定有效期；必须过期之后才可以取出来 peek(),poll(),take(),
  • 元素要实现 Delayed 接口 ：long getDelay(TimeUnit unit) 返回剩余时间，如果小于等于0 表示过期。有序，PriorityQueue (数组结构)

原理：

  1. 内部定义一个优先级队列，使用完全二叉堆实现排序
  2. 取出元素：如果没有达到延时时间，就延时堵塞，到时间会自动醒来。
  3. 设置leader：优化阻塞通知，减少不必要的等待时间。

总结

leader：指向正在取数据的线程

  1. 取数据：先判断有没有数据（等待），再看时间没到到，没到就查看leader，
    • 如果不为空：说明已经有线程在取了，自己自然取不出，就await等待
    • 如果为空：leader指向自己，awaitNanos()延迟等待自动唤醒。
  2. 存数据：如果存入的数据优先级最高，就表示没有能取的数据，把leader设置为null，唤醒取一个数据的线程。

几种情况

  • take但没有数据被堵塞：添加数据的时候会被唤醒成功取出数据
  • take 但是 leader不为空 被堵塞：leader指向的数据 是awaitNanos()延迟等待，自动醒来后会唤醒当前的线程。

leader的作用：减少不必要的等待时间，只要为非nul，说明有线程再取且在等待，就说明自己肯定也取不出，直接堵塞。

7、阻塞队列-SynchronousQueue

特点：

  • 最多放一个数据，存了就必须去取出来在存，即大小为1的堵塞队列
  • 线程A put( ) 存东西之后会立即进入堵塞状态。此时只有线程B进行 take( ) 取操作才能让线程A从堵塞状态继续运行。如果A堵塞后，B也是取，那么B也会堵塞
  • 处于堵塞状态时其他API都不能调用，只能使用take
  • 默认非公平的，可以设置为公平的
  • 性能高
