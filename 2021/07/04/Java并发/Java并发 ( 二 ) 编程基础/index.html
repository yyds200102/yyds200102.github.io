1、线程简介

1.1、什么是线程

​ 操作系统运行一个程序时，会为其创建一个进程。操作系统调度的最小单元是线程，也叫轻量级进程，一个进程可以创建多个线程，每个线程有自己的计数器、栈、局部变量，能够访问共享的方法区、堆。处理器在线程上高速切换，让使用者感觉线程在同时执行。

​ 一个Java程序就有多个线程，至少有主线程、GC线程，异常处理线程。

程序、进程、线程

  • 程序：指一段静态的代码，静态对象。
  • 进程：程序的一次执行过程，或是正在运行的一个程序。
  • 线程：进程可进一步细化为线程，是一个程序内部的一条执行路径。

进程属于操作系统的范畴，主要是同⼀段时间内，可以同时执行⼀个以上的程序，而线程则是在同⼀程序内几乎同时执行⼀个以上的程序段

并行与并发

  • 并行：多个线程同时执行多个任务。
  • 并发：多个线程同时执行一个任务。

1.2、多线程的好处

更多的处理核心

处理器上的核心越来越多，超线程技术运用广泛，现在大多数计算机更加擅长并行计算

更快的响应时间

把数据性不强的操作派发给其他线程处理（也可以使用消息队列）这样响应用户请求的线程能够尽可能地处理完成，缩短了响应时间，提示用户体验。

更好的编程模型

Java提供的多线程编程的良好环境

3、线程的终止

3.1、中断

概念：中断可以理解为线程的一个标识位，标识一个运行中的线程是否被其他线程进行了中断操作。线程内部通过检查自己是否被中断来进行响应。

作用：让处于堵塞状态的线程停止一次堵塞（睡眠的抛出异常并停止睡眠，等待的抛出异常并停止等待），对没有处于中断堵塞状态的线程没有效果。

  • 等待堵塞（wait）、睡眠、IO堵塞、同步堵塞（synchorized）、

本质：比如Thread.sleep 需要捕获异常，如果设置中断，就会抛出异常并且被捕获，程序会重捕获的代码之后继续运行； synchorized等待的线程设置中断没有效果。

4.5、join的使用

线程A调用thread.join()：表示线程A开始等待，直到线程thread执行完成后，线程A才继续执行。

原理：假设线程A调用 thread.join 这个同步方法，那么线程A就持有了thread对象的锁，所以调用wait 方法让线程A一直等待，等到线程thread 结束之后，JVM会调用 notify 让线程A苏醒。

总之：就是通过wait方法让线程等待，JVM负责唤醒

4.6、ThreadLocal的使用

ThreadLocal相当于一个map：所有线程的线程ID作为map的键，值作为value，不同线程设置的值和获得的值是不同的，互相没有冲突。

原理：内部维护了一个ThreadLocalMap（节点Entry），每个线程的ID作为键，存的数据作为value

内存泄漏问题：内部map的key 是弱引用，value是强引用，所以可能 key 被回收了，但是value 永远无法回首，ThreadLocal 解决的思路是：调用3个方法的时候，会自动清除键为 null 的 Entry 结构。

4.7、Sleep 和 wait的异同

同：

  • 两者都可以暂停线程的执行。

异：

  • sleep 不会 释放锁，wait会释放锁
  • sleep 会 自动苏醒，wait需要其他线程唤醒或者超时等待自动苏醒

5、死锁探究

原因：两个线程互相占用对方需要的资源

  • 顾客不先付钱（锁住use），并尝试获取试用（获取锁buy），不然不付钱（释放锁use）
  • 老板不让试用（锁住buy），并尝试先获得钱（获取锁use），不然不让试用（释放锁buy）

顾客和老板各自拥有自己的锁，不释放给对方，就这样一致互相等待对方。

避免死锁

  • 破坏互斥条件 ：这个条件我们没有办法破坏，因为我们⽤锁本来就是想让他们互斥的（临界资源需要互斥访问）。
  • 破坏请求与保持条件 ：⼀次性申请所有的资源。
  • 破坏不剥夺条件 ：占⽤部分资源的线程进⼀步申请其他资源时，如果申请不到，可以主动释放它占有的资源。
  • 破坏循环等待条件 ：靠按序申请资源来预防。按某⼀顺序申请资源，释放资源则反序释放。破坏循环等待条件。
