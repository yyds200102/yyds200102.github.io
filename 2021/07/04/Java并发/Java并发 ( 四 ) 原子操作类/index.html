1、CAS和unsafe

1.1、CAS

概念：

  1. CAS 即 compareAndSet
  2. 比较当前**工作内存中的值和主内存**的值，
  3. 如果是期望的，那么执行操作如果不是就一直循环。

获取共享变量时，需要**使用volatile保证可见性，CAS和volatile可以实现无锁并发**，适用于竞争不激烈的情况

  • 竞争不激烈，效率高
  • 竞争激烈，循环次数多，反而影响效率

CAS底层依赖于一个Unsafe类来直接调用系统底层的CAS指令

2、原子类概述

概念：原⼦类说简单点就是具有原⼦/原⼦操作特征的类。

介绍

  • 基于 CAS+unsafe+volatile，可以多线程情况下不加锁，仍然保持线程安全

  • unsafe：操作c,c++库，jni操作，发送指令通过硬件资源，直接操作内存，原子操作，不安全危险

思想：通过自旋CAS实现的

缺点：

  1. 循环耗时、
  2. 一次只能保证一个共享变量原子性
  3. 存在ABA等问题

分类：

  • 基本类型：AtomicInteger，AtomicBoolean，AtomicLong

  • 数组类型：AtomicLongArray，AtomicIntegerArray，AtomicReferenceArray

  • 引用类型：AtomicReference，AtomicStampedReference，AtomicMarkableReference 、

  • 字段类型：AtomicIntegerFieldUpdater，AtomicLongFieldUpdater，AtomicReferenceFieldUpdater，

  • 累加器：LongAdder，DoubleAdder，LongAccumulator，DoubleAccumulator
