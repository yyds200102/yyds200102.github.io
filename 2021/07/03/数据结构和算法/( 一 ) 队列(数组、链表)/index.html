3、队列（数组）

3.1、介绍

  1. 队列是一个有序列表，可以用数组或是链表来实现。
  2. 遵循先入先出的原则，进入都是在队尾，出去都是在队首

3.2、数组实现

  1. 因此需要两个变量 front及 rear分别记录队列前后端的下标，arr[ 0 ]作为队首
  2. front 会随着数据输出而改变，初始为 - 1 ，每次操作时先自增，然后 返回 arr[ front ]
  3. rear则是随着数据输入而改变，初始为 - 1 ，每次操作时先 自增，然后操作 arr[ rear ]

注意：

  • 出队列逻辑上删除，实际并没有删除，通过模拟指针右移，这样指针左边的空间会浪费掉
  • 如上述，添加3次数据后，即使把数据全删了，也无法继续添加数据

解决：环形数组

3.3、环形数组

思路：

  • 若rear指针已经到了数组尽头，且数组没满，则让rear重新指向数组的第一位

注意：

  1. 此时输入3，只能存入2个数据，为什么？如果能存入3个数据，那么满和空的条件一致。

  2. 所以是判断存了2个数据，就满了，所以只要判断，rear的下一个地址和front相等即可

    (rear + 1 - front) % maxSize == front 分析：

    • 情况一： rear + 1 = front，这种情况就是说 rear 下一个地址，就是front
    • 情况二：rear + 1 - maxSize = front ，当rear为数组末尾时，下一个地址循环，减去maxSize

  3. 有效个数等于 front - rear ，如果 rear已经循环了一次在front前面，则加上maxSize

    (rear + maxSize - front) % maxSize

2、队列（链表）

4.1、单链表介绍

  1. 链表是以节点的方式来存储,是链式存储，每个节点包含 data 域， next 域：
  2. 链表的各个节点不一定是连续存储.
  3. 链表分带头节点的链表和没有头节点的链表，根据实际的需求来确定

4.2、单链表题

1、单链表的反转

  • 很容易想到存放在数组里然后重新构建链表，但是效率很低
  • 头插法，效率更高

2、从尾到头打印单链表

方法：

  1. 先反转，然后打印，会改变链表的结构，不可取
  2. 利用栈的特点，先进后出打印
  3. 存到数组、list集合里，然后输出，效率低

3、合并两个有序的单链表，合并之后的链表依然有序

  • 可以把一个单链表分割成许多节点，依次插入另一个单链表
  • 新建链表，每次插入前比较两个链表谁的更小，小就插入

4.4、单向环形链表

约瑟夫问题（约瑟夫环）

设编号为1，2，… n的n个人围坐一圈，约定编号为k（1<=k<=n）的人从1开始报数，数到m 的那个人出列，它的下一位又从1开始报数，数到m的那个人又出列，依次类推，直到所有人出列为止，由此产生一个出队编号的序列。

思路：

  1. 需求创建一个辅助指针(变量) helper , 事先应该指向环形链表的最后这个节点.
  2. 编号为k，则helper和first（指向链头）右移 k - 1 次，然后移动m - 1 次
  3. 需要出列时， helper = first.next 即可
  4. 重复直到所有人出列

4.4、队列（链表实现）

思路：

  • 链表的头结点当做队尾，插入节点使用头插法
  • 链表的尾部当做队首，出队列在链表的尾部进行
