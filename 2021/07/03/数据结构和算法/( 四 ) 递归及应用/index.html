数据结构和算法(四):递归

1、概念

基本概念：

递归就是方法自己调用自己,每次调用时传入不同的变量.递归有助于编程者解决复杂的问题,同时可以让代码变得简洁

递归调用规则：

  1. 当程序执行到一个方法时，就会开辟一个独立的空间(栈)

  2. 每个空间的数据(局部变量)，是独立的.

  • 循环打印直到2
  • 阶乘

3、递归规则

  1. 执行一个方法时，就创建一个新的受保护的独立空间(栈空间)
  2. 方法的局部变量是独立的，不会相互影响, 比如n变量
  3. 如果方法中使用的是引用类型变量(比如数组)，就会共享该引用类型的数据.
  4. 递归必须向退出递归的条件逼近，否则就是无限递归,出现StackOverflowError，死龟了:)
  5. 当一个方法执行完毕，或者遇到return，就会返回，遵守谁调用，就将结果返回给谁，同时当方法执行完毕或者返回时，该方法也就执行完毕。

4、迷宫问题

  1. 小球得到的路径，和程序员设置的找路策略有关即：找路的上下左右的顺序相关
  2. 再得到小球路径时，可以先使用(下右上左)，再改成(上右下左)，看看路径是不是有变化
  3. 测试回溯现象

思路：

  1. 小球在每一个点都按照下右上左的顺序走，走过的点设置为2，且走过的路不再走
  2. 如果过程中map[6] [5] 为2，那么表示该路到达了终点，返回true；
  3. 如果走到某一点上、下、左、右全部都走不通，或已经走过，那么表示找不到路
  4. 所以走的顺序就是，一直往下走，走不了先往右，还是走不了就往上，还是不行再往左，直到成功

5、八皇后问题

思路：

  1. 用数组表示棋盘，数组索引表示每一列，值表示皇后在该列的位置
  2. 把每次往数组中添加值当做放皇后，arr[0] 就是第一个皇后，值就是第一个皇后位置
    • 每次放皇后若成功，即arr[i] = value 且不与前面的冲突，则往后一列 arr[i+1] =0 并且往该列的下一个位置 arr[i] = value + 1 尝试；若失败，只往该列下一行尝试 arr[i] = value + 1
    • 一直放到最后一行，可以依次判断最后一行的每个位置是否能放，如果能放就返回1，否则返回0

简化思路：

  1. 用数组表示棋盘，数组索引表示每一列，值表示皇后在该列的位置
  2. 每次判断防止第n个皇后，在第n列的8行能不能放下，如果可以，就去下一列继续判断
  3. 相比于上面，少了每列中的递归，抽成for循环了
